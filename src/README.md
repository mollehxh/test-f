# Архитектура

Проект использует минимальный вариант Feature-Sliced Design:

- `app` — точка сборки приложения, глобальные провайдеры и стили;
- `pages` — полноценные экраны приложения;
- `shared` — UI-kit и технические утилиты без бизнес-логики.

Слоёв `widgets`, `features` и `entities` сейчас нет намеренно. Их следует
добавлять только при появлении реальной отдельной ответственности или
переиспользуемой бизнес-логики.

## Текущие страницы

- `pages/main` — основной экран с заводами, графиком и операциями;
- `pages/settings` — самостоятельный экран настроек.

## Правила импортов

1. Зависимости направлены вниз: `app` → `pages` → `shared`.
2. `app` импортирует страницы через их `index.ts`.
3. Файлы внутри одной страницы используют относительные импорты.
4. Одна страница не импортирует внутренности другой страницы.
5. `shared` не зависит от `pages` или `app`.
